# SD1.5 Sequential 2-Pass (SDEdit + ControlNet) Experiment

> FlowEdit + CN의 극단적 trade-off를 해결하기 위해
> SD1.5 img2img 기반의 Sequential 2-pass ControlNet 파이프라인으로 전환한 실험 기록

---

## Overview

FlowEdit + ControlNet은 CN 강도 조절에 따라
- 너무 약하면 조건 반영이 안 되고
- 너무 강하면 이미지가 붕괴되는 문제가 있었음.

이를 해결하기 위해 SD1.5 img2img + ControlNet으로 전환하고,
condition interference를 줄이기 위해 2-pass sequential 구조를 사용.

목표:
1. Pass 1에서 잇몸/치아 경계 교정 (segmentation)
2. Pass 2에서 표면 디테일 보강 (canny/depth 계열)

---

## Method

### Sequential 2-Pass

`noisy render -> Pass1(seg) -> pass1 output -> Pass2(detail) -> final output`

- Pass 1: `control_v11p_sd15_seg`
- Pass 2: 후보 비교 후 최종 `control_v11p_sd15_canny` 채택

### Segmentation 색상 매핑 (ADE20K)

`control_v11p_sd15_seg`는 ADE20K 색상 분포에 맞춘 입력이 필요.

```python
ADE_BACKGROUND = (120, 120, 120)
ADE_GUM        = (150,   5,  61)
ADE_TOOTH      = (230, 230, 230)
```

### Normal -> Edge 변환 (Pass2 canny condition)

```python
def normal_to_edges(normal_pil, target_size, threshold=30, dilate=1):
    gray = normal_pil.resize(target_size).convert("L")
    edges = gray.filter(ImageFilter.FIND_EDGES)
    edges_arr = np.where(np.array(edges) > threshold, 255, 0).astype(np.uint8)
    edges = Image.fromarray(edges_arr, "L")
    if dilate > 0:
        edges = edges.filter(ImageFilter.MaxFilter(dilate * 2 + 1))
    return edges.convert("RGB")
```

## Experimental Setup

### 데이터 경로

| 항목 | 경로 |
|---|---|
| Input noisy render | `LoRA/dataset/test/ori/` |
| Segmentation | `data/tooth/segmentation/` |
| Normal | `data/tooth/normals/` |
| Depth | `data/tooth/depths/` |

### 주요 스크립트

| 파일 | 설명 |
|---|---|
| `SDEdit/refine_sdedit.py` | FLUX dual CN 초기 버전 |
| `SDEdit/refine_sequential.py` | FLUX sequential 버전 |
| `SDEdit/refine_sd_sequential.py` | SD1.5 sequential 메인 |

### 실행 명령

```bash
conda run -n skyfall-gs_copy python SDEdit/refine_sd_sequential.py \
    --input_dir  LoRA/dataset/test/ori \
    --normal_dir data/tooth/normals \
    --seg_dir    data/tooth/segmentation \
    --output_dir SDEdit/results/sd_sequential \
    --save_pass1
```

### 주요 인자

| 인자 | 값 | 설명 |
|---|---|---|
| `--seg_strength` | `0.3` | Pass1 img2img strength |
| `--seg_scale` | `0.9` | Pass1 Seg CN 강도 |
| `--detail_strength` | `0.2` | Pass2 img2img strength |
| `--detail_scale` | `0.4` | Pass2 Detail CN 강도 |
| `--guidance_scale` | `2.0` | CFG (img2img에서는 낮은 값이 안정적) |
| `--save_pass1` | `False` | Pass1 중간 결과 저장 |

## Results

### Pass2 조건 실험 로그

| 시도 | 조건 | 결과 | 결론 |
|---|---|---|---|
| 1 | `control_v11p_sd15_normalbae` | normal 분포/좌표계 불일치로 품질 붕괴 | 중단 |
| 2 | `control_v11f1p_sd15_depth` | depth 정확도/정합성 부족 | 중단 |
| 3 | `control_v11f1e_sd15_tile` | 구조 강화 효과 약함 | 중단 |
| 4 | `control_v11p_sd15_canny` (normal-derived edge) | 경계 정보 반영 안정적 | 채택 |

### 하이퍼파라미터 핵심 발견

| 파라미터 | 비권장 | 권장 | 해석 |
|---|---|---|---|
| `guidance_scale` | `7.5` | `2.0` | 높은 CFG는 텍스트 쏠림으로 구조 정합성 저하 |
| `seg_strength` | `0.1` | `0.15~0.3` | 너무 낮으면 step 부족으로 CN 효과 약화 |
| `detail_strength` | `0.1` | `0.2` | 너무 낮으면 디테일 조건 반영 불충분 |

## Analysis

Sequential 2-pass는 동시 multi-condition 대비 간섭이 줄어,
경계 교정과 디테일 보강을 분리해서 제어할 수 있었음.

현재 판단:
- Pass1(seg)로 구조 안정화
- Pass2(canny-from-normal)로 디테일 반영
- FlowEdit + CN보다 제어 가능성과 재현성이 높음

## Troubleshooting

### cv2 / NumPy 충돌

`AttributeError: _ARRAY_API not found`

해결:
- cv2 의존 제거
- PIL `ImageFilter.FIND_EDGES` + `MaxFilter`로 edge 생성 대체

### HuggingFace repo ID 오타

`lllyasviel/control_v11p_sd15_depth` -> 잘못된 ID

해결:
- 정확한 depth CN ID 사용: `control_v11f1p_sd15_depth`
