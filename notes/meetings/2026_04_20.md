## Sequential → Global Optimization

### 문제 정의
현재 방식은 view별 **순차적(local) 최적화** 구조

현재 방식 :
view1 → optimize → freeze
view2 → optimize → freeze
view3 → optimize

- visibility 기반 freeze mask로 **consistency는 유지**
- 그러나 **초기 오류가 이후 단계에서 수정되지 않는 구조적 한계 존재**
  - 예: 정면→오른쪽 학습 후, 이후 view에서 기존 영역 수정 불가

---

### 해결 방안

- **Stage 1 (Sequential)**: 빠르게 초기 해 생성 (warm-up)
- **Stage 2 (Global)**: 모든 view를 동시에 사용하는 joint optimization

즉, sequential 결과를 초기값으로 사용 → global에서 전체 일관성 재최적화

---

## Stage 1: Sequential Optimization

- 5개 대표 view 사용 (sparse sampling)
- 이전 view의 texture를 초기값으로 전달하며 순차 최적화
- visibility 기반 freeze: 해당 view에서 **보이지 않는 UV 텍셀은 gradient 차단**
- 빠르게 합리적인 초기 texture 생성
- global optimization의 안정적인 시작점 확보
  

### View 구성 전략
- dense view 대신 **간격 있는 view 선택**
- 이유: 불필요한 freeze 영역 최소화

#### 기존: 24-view 구성

- 시작: (elevation -30°, azimuth 0°)
- azimuth + 방향으로 15° 간격 순차 확장
- 다시 기준 뷰 복귀 후 azimuth - 방향 동일 확장
- 이후 (elevation 45°, azimuth 0°) 기준으로  
  elevation -15° 간격으로 확장

→ 전체적으로 **촘촘한 ring + elevation sweep 구조**

#### 수정: 5-view 구성

다음 5개 view만 사용:

1. (elevation -30°, azimuth 0°)
2. (elevation -30°, azimuth +75°)
3. (elevation -30°, azimuth -75°)
4. (elevation 15°,  azimuth 0°)
5. (elevation -75°, azimuth 0°)

→ azimuth 방향 2개 + elevation 방향 2개로 최소 구성

## Stage 2: Global Optimization

- 24개 view **동시 사용**
- 매 iteration:
  - 모든 view 렌더링
  - loss 합산
  - **single backward**

- Stage 1 결과를 초기값으로 사용
- view 간 충돌을 전체적으로 조정
- 초기 freeze로 고정된 오류를 재조정 가능

+두 스테이지 모두 raw 렌더 픽셀값이 아닌 **albedo 추정값**을 비교
