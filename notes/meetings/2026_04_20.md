Sequential to Global optimization 

현재 방식 :
view1 → optimize → freeze
view2 → optimize → freeze
view3 → optimize
→ 순차적으로(local)만 맞추는 구조

visibility 기반 freeze mask는 consistency는 유지하지만, 초기 오류가 이후 단계에서 **수정되지 않는 trade-off** 존재 
(예: 정면→오른쪽을 먼저 학습한 뒤, 왼쪽/하단 view를 학습해도 기존 영역이 그대로 유지됨)

해결 방안 : 초기 sequential로 빠르게 근사값 생성. 이후, global optimization (bundle-like).
즉, 앞에서 만든 순차적 학습을 global 최적화의 warm-up으로 씀. 그 후, 여러 뷰를 동시에 보고, 하나의 공통 변수를 joint optimization, 모든 view 기준으로 동시에 loss 걸어서 다시 최적화

- stage1(sequential) : 지금 하고 있는 방식대로 view 순서대로 inverse rendering => 초기 texture 생성
- stage2(global) : 모든 view를 동시에 사용

+ 우선, 치아가 주로 보이는 뷰로 국한, 치아와 실제 스캔 부분의 잇몸만 최적화 대상으로, 그 외 만들어진 부분은 최적화에서 제외

1. 일단 Json 파일 간소화 되도록 재구성 - 앞서 실험해보니, 오히려 촘촘하게 뷰를 구성하는 거보다 조금은 떨어져있는 뷰로 구성하는게 낫다고 생각이 들었음(왜냐면 쓸데없이 프로즌 되는 부분을 최소화하기 위해) => elevation -30도에서 Azimuth 0도 75도 -75도, Azimuth0도에서 elevation -30도, 15도, -75도 이렇게 구성

2. stage1(sequential)
위 방식 그대로 사용
- 5개 대표 뷰를 순서대로 처리
- 각 뷰에서 이전 뷰의 텍스처를 초기값으로 받아 최적화
- 목적: 합리적인 초기 텍스처를 빠르게 만드는 것 (global 최적화의 warm-up)
- Visibility freeze: 각 뷰에서 보이지 않는 UV 텍셀은 gradient를 받지 않도록 차단

3. stage2(global)
24개 뷰를 **동시에** 하나의 loss로 묶어서 최적화
매 iteration마다 24개 뷰를 전부 렌더링하고, 각 loss를 합산한 뒤 **단 한 번의 backward**
Stage 1의 최종 텍스처를 시작점으로 사용

+두 스테이지 모두 raw 렌더 픽셀값이 아닌 **albedo 추정값**을 비교
