# Inverse Gather Texture Baking

## 1. Inverse gather

**Forward bake** (texel → screen):
- 각 UV 텍셀의 3D 위치를 카메라 공간으로 투영 → 화면 좌표에서 색상 샘플링
- 가시성 판단을 위한 UV match 검사 과정에서 허용 반경(neighborhood) 내 최소 거리를 허용 → false positive 발생 가능

**Inverse gather** (screen → texel):
- AOV 렌더가 각 화면 픽셀의 UV 좌표를 직접 출력
- AOV가 현재 visible surface의 UV를 기록하므로, forward 방식의 local UV matching 기반 false positive를 줄일 수 있음
- UV 역변환: `tx = round(u × TW)`, `ty = round((1−v) × TH)` (H2 convention)
- UV match 검사 없이 visible screen pixel의 UV를 직접 texel로 매핑
- local UV matching 기반 false positive 감소

## 2. 전체 처리 흐름

#### 사전 준비
- OBJ 메시 파싱
- nvdiffrast로 UV 공간 래스터라이즈
- 각 UV 텍셀에 대응하는 3D 월드 좌표(`pos3d_tex`) 계산
- 메시 마스크(`mesh_mask`) 생성

### 뷰 반복 loop

각 뷰에 대해

#### 1. 현재 누적 텍스처 렌더 (pre-gather render)
- Mitsuba로 현재 누적 texture render
- 현재 상태의 화면 이미지 생성

#### 2. 타겟 이미지 결정
- 첫 뷰 : 사전 생성된 Gemini 보정 이미지 직접 사용
- 이후 뷰 : 렌더 결과 → Gemini 보정 → 보정 이미지를 target으로 사용

#### 3. Inverse gather
- AOV UV 렌더로 각 화면 픽셀의 `(u, v)` 좌표 획득

- valid 조건:
  - `AOV alpha > 0.5`
    : 현재 screen pixel에 실제 mesh surface가 렌더된 경우만 사용
  - `target alpha > 0.5`
    : target image에서 유효한 픽셀만 사용
  - UV→texel 결과가 mesh 영역 내부인지

- UV 역변환:
  - `tx = round(u × TW)`
  - `ty = round((1−v) × TH)` (H2 convention)

- 여러 screen pixel이 동일 texel로 quantize될 수 있으므로, view 내부 평균 RGB 계산 후 누적
  - `accum[ty, tx] += mean_rgb`
  - `weight[ty, tx] += 1.0`

#### 4. 중간 텍스처 갱신 (EXR 저장)
- geometry-space texture를 bitmap/render-space로 vertical flip 후 저장 -> 다음 뷰 렌더에 사용

#### 5. 사후 검증 렌더 (post-gather render)
- 동일 뷰 재렌더를 통해 gather 결과 consistency 확인


#### 결과
- 커버된 texel만 평균화
- 미커버 texel : magenta 유지
