# Texture Baking: Splatting vs Texel-Centric Gathering

## 1. Splatting vs Gathering

### Splatting
- screen pixel을 기준으로 시작
- AOV UV를 이용해 screen pixel이 어느 UV texel에 대응되는지 계산
- `tx = round(u × TW)`, `ty = round((1−v) × TH)`로 destination texel 결정
- 해당 screen pixel의 RGB를 texel에 누적
흐름:
`screen pixel → AOV UV → rounded texel → accum/weight update`

### Texel-centric gathering
- UV texel을 기준으로 시작
- 각 texel의 3D 위치(`pos3d_tex`)를 camera로 projection
- projected screen coordinate `(px, py)`에서 target RGB를 bilinear sampling
- sampled RGB를 동일 texel `(ty, tx)`에 기록
흐름:
`texel → pos3d_tex → camera projection → bilinear sampling → same texel update`

## splatting 방식 처리 흐름(이전 시도 방식)

#### 사전 준비
- OBJ 메시 파싱
- nvdiffrast로 UV 공간 래스터라이즈
- 각 UV 텍셀에 대응하는 3D 월드 좌표(`pos3d_tex`) 계산
- 메시 마스크(`mesh_mask`) 생성

### 뷰 반복 loop

각 뷰에 대해

#### 1. 현재 누적 텍스처 렌더 (pre-splat render)
- Mitsuba로 현재 누적 texture render
- 현재 상태의 화면 이미지 생성

#### 2. 타겟 이미지 결정
- 첫 뷰 : 사전 생성된 Gemini 보정 이미지 직접 사용
- 이후 뷰 : 렌더 결과 → Gemini 보정 → 보정 이미지를 target으로 사용

#### 3. splatting
- AOV UV 렌더로 각 화면 픽셀의 `(u, v)` 좌표 획득

- valid 조건:
  - `AOV coverage > 0.5`
    : 현재 screen pixel에 실제 mesh surface가 렌더된 경우만 사용
  - `target coverage > 0.5`
    : target image에서 유효한 픽셀만 사용
  - UV→texel 결과가 mesh 영역 내부인지

- UV 역변환:
  - `tx = round(u × TW)`
  - `ty = round((1−v) × TH)` (H2 convention)

- 여러 screen pixel이 동일 texel로 quantize될 수 있으므로, view 내부 평균 RGB 계산 후 누적
  - `accum[ty, tx] += mean_rgb`
  - `weight[ty, tx] += 1.0`

#### 4. 중간 텍스처 갱신 (EXR 저장)
- geometry-space texture를 bitmap/render-space로 vertical flip 후 저장 -> 다음 뷰 렌더에 사용

#### 5. 사후 검증 렌더 
- 동일 뷰 재렌더를 통해 splatting 결과 확인


#### 결과
- 커버된 texel만 평균화
- 미커버 texel : magenta 유지 => 미커버 영역 Post-processing 제거

## Gathering 처리 흐름(현재 방식)

#### 사전 준비
- OBJ mesh를 파싱하여 vertex position, UV, face 정보 로드
- nvdiffrast로 UV space를 rasterize
- 각 UV texel에 대응되는 3D 위치 `pos3d_tex` 계산 (`pos3d_tex[ty, tx]`는 UV texel `(tx, ty)`가 실제 mesh surface의 어느 3D 위치에 해당하는지를 저장한 값, 2번에서 사용)
- mesh에 속하는 texel만 표시하는 `mesh_mask` 생성(2번에서 사용)
- texture accumulation용 배열 초기화
  - `accum_np`: texel별 누적 RGB
  - `weight_np`: texel별 누적 횟수
  - unseen texel(`weight_np==0`)은 magenta 유지

### 뷰 반복 loop 

각 뷰에 대해

#### 1. AOV UV 렌더
- 현재 camera view에서 3D mesh를 일반 렌더처럼 screen/image plane에 렌더
- 단, RGB 색상을 저장하는 것이 아니라 각 screen pixel에 대해 다음 정보를 저장
  - `aov_coverage`: screen-space foreground mask의 coverage 값, mesh 내부는 1, background는 0이며, silhouette 경계는 0~1 사이
  - `aov_u`, `aov_v`: 해당 screen pixel이 보고 있는 surface의 UV 좌표

색을 기록할 texel을 정하기 위함이 아닌, 2번 단계에서 각 UV texel을 screen으로 projection한 뒤, 3번 단계에서 그 projected 위치가 실제 mesh를 보고 있는지, 그리고 현재 texel과 같은 표면을 보고 있는지 확인하기 위해 사용함

#### 2. Texel-centric gather
- 현재 방식은 UV texel 선택 → 3D 위치 projection → screen에서 색 sampling → 같은 texel에 누적하는 방식 사용
- UV texel 후보 선택 하기 위해 사전 준비한 `mesh_mask` / `pos3d_tex` 기준으로 mesh 위에 있는 texel 후보 선택
- 각 texel에 대해:
  - `pos3d_tex[ty, tx]`를 현재 camera view로 projection
  - projected screen coordinate `(px, py)` 계산
  - `(px, py)`가 image bounds 내부인지 확인
이 과정에서 선택된 texel (ty, tx)가 destination texel

#### 3. Visibility / correspondence 검증
2번에서 texel을 screen으로 projection해도, 그 screen pixel이 실제로 현재 texel을 보고 있다고 바로 볼 수는 없음 따라서 projected pixel 위치에서 1번의 AOV 정보를 읽어 다음을 확인
- `aov_coverage > 0.5` : pixel이 background가 아니라 mesh 영역인지 확인 mesh 내부는 1, background는 0이며, silhouette 경계는 0~1 사이
- `target_coverage > 0.5` : 색을 가져올 target image(gemini image)에서도 해당 pixel이 background가 아니라 object 영역인지 확인. object영역은 1, background는 0
- projected pixel의 AOV UV가 현재 texel과 strict nearest 기준으로 일치하는지(pixel이 단순히 mesh를 보고 있는 것뿐 아니라, 현재 texel과 같은 표면을 보고 있는지 확인)

따라서 해당 조건을 통과하지 못하면, screen에서는 mesh가 보이더라도 현재 texel에는 색을 기록하지 않음

> 현재 방식은 projected pixel의 AOV UV와 현재 texel UV를 strict nearest 기준으로 비교. 이 방식은 screen pixel과 texture texel 사이의 해상도 차이 및 mapping footprint를 고려하지 못하기 때문에, minification / magnification 상황에서 sampling issue가 발생할 수 있음. 자세한 내용은 [## 4. 피드백](#4-피드백)에 정리

#### 4. Target RGB sampling
- 위 조건을 모두 통과한 texel에 대해서만 target image에서 RGB 값을 가져옴.
- 이때 nearest pixel 하나를 그대로 가져오는 것이 아니라, `(px, py)`를 기준으로 주변 4개 pixel에서 bilinear sampling

#### 5. Texture accumulation
- sampling한 RGB 값을 현재 texel에 누적
  - `accum_np[ty, tx] += sampled_rgb` : 누적값 저장
  - `weight_np[ty, tx] += 1.0` : 몇 번 누적되었는지 저장

#### 6. 중간 텍스처 갱신 및 post-gather render
- geometry-space texture를 bitmap/render-space로 vertical flip 후 저장 -> 다음 뷰 렌더에 사용
- 동일 뷰 재렌더를 통해 gather 결과 확인


#### 결과
- 모든 view 처리 후 texel별 평균 계산
  - `baked = accum_np / weight_np`
- `weight_np == 0`인 미커버 texel은 magenta 유지 => 미커버 영역 Post-processing 제거

## 4. 피드백 
현재 구현에서는 texel을 screen으로 projection한 뒤, projected pixel에서 읽은 AOV UV를 texture 해상도 기준 좌표로 변환하고, 이 값이 현재 texel 좌표와 strict nearest 기준으로 가까운지 확인하는 방식 사용  
즉, projected pixel에서 실제로 보이는 surface의 UV 위치와 현재 gather 중인 texel의 UV 위치 비교  
그러나 해당 방법은 screen pixel 과 texture texel 의 맵핑 및 해상도 차이를 고려하지 못하여, 샘플링 이슈가 반드시 발생(minification / magnification)  

스크린에서 보이는데 UV 에서 마젠타가 나오는 이유 원인 파악하기 위해  
`weight_np == 0`으로 남은 texel에 대해 원인별 색상 표시를 추가하였음.
원인별 표시 색상:
- black: mesh에 속하지 않는 texel
- gray: camera projection 실패 또는 image bounds 밖
- blue: AOV 기준 mesh coverage 조건 실패
- orange: target image 기준 object coverage 조건 실패
- red: projected pixel의 AOV UV와 현재 texel UV가 strict nearest 기준으로 일치하지 않아 texture update에서 제외된 texel
- green: texture update에 사용된 texel

=> Z-buffer Visibility 사용하여 현재 texel의 3D 위치가 해당 view에서 실제로 보이는 surface인지 판단하는 방향으로 수정
