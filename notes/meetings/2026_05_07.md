# Texture Baking: Splatting vs Texel-Centric Gathering

## 1. Splatting vs Gathering

### Splatting
- screen pixel을 기준으로 시작
- AOV UV를 이용해 screen pixel이 어느 UV texel에 대응되는지 계산
- `tx = round(u × TW)`, `ty = round((1−v) × TH)`로 destination texel 결정
- 해당 screen pixel의 RGB를 texel에 누적
흐름:
`screen pixel → AOV UV → rounded texel → accum/weight update`

### Texel-centric gathering
- UV texel을 기준으로 시작
- 각 texel의 3D 위치(`pos3d_tex`)를 camera로 projection
- projected screen coordinate `(px, py)`에서 target RGB를 bilinear sampling
- sampled RGB를 동일 texel `(ty, tx)`에 기록
흐름:
`texel → pos3d_tex → camera projection → bilinear sampling → same texel update`

## splatting 방식 처리 흐름

#### 사전 준비
- OBJ 메시 파싱
- nvdiffrast로 UV 공간 래스터라이즈
- 각 UV 텍셀에 대응하는 3D 월드 좌표(`pos3d_tex`) 계산
- 메시 마스크(`mesh_mask`) 생성

### 뷰 반복 loop

각 뷰에 대해

#### 1. 현재 누적 텍스처 렌더 (pre-gather render)
- Mitsuba로 현재 누적 texture render
- 현재 상태의 화면 이미지 생성

#### 2. 타겟 이미지 결정
- 첫 뷰 : 사전 생성된 Gemini 보정 이미지 직접 사용
- 이후 뷰 : 렌더 결과 → Gemini 보정 → 보정 이미지를 target으로 사용

#### 3. Inverse gather
- AOV UV 렌더로 각 화면 픽셀의 `(u, v)` 좌표 획득

- valid 조건:
  - `AOV alpha > 0.5`
    : 현재 screen pixel에 실제 mesh surface가 렌더된 경우만 사용
  - `target alpha > 0.5`
    : target image에서 유효한 픽셀만 사용
  - UV→texel 결과가 mesh 영역 내부인지

- UV 역변환:
  - `tx = round(u × TW)`
  - `ty = round((1−v) × TH)` (H2 convention)

- 여러 screen pixel이 동일 texel로 quantize될 수 있으므로, view 내부 평균 RGB 계산 후 누적
  - `accum[ty, tx] += mean_rgb`
  - `weight[ty, tx] += 1.0`

#### 4. 중간 텍스처 갱신 (EXR 저장)
- geometry-space texture를 bitmap/render-space로 vertical flip 후 저장 -> 다음 뷰 렌더에 사용

#### 5. 사후 검증 렌더 (post-gather render)
- 동일 뷰 재렌더를 통해 gather 결과 consistency 확인


#### 결과
- 커버된 texel만 평균화
- 미커버 texel : magenta 유지

## Gathering 처리 흐름

#### 사전 준비
- OBJ mesh를 파싱하여 vertex position, UV, face 정보 로드
- nvdiffrast로 UV space를 rasterize
- 각 UV texel에 대응되는 3D 위치 `pos3d_tex` 계산
- mesh에 속하는 texel만 표시하는 `mesh_mask` 생성
- texture accumulation용 배열 초기화
  - `accum_np`: texel별 누적 RGB
  - `weight_np`: texel별 누적 횟수
  - unseen texel은 magenta 유지

### 뷰 반복 loop 

각 뷰에 대해

#### 1. AOV UV 렌더
- Mitsuba AOV 렌더로 현재 view의 per-pixel 정보를 획득
  - `aov_alpha`: 해당 screen pixel에 mesh surface가 렌더되었는지
  - `aov_u`, `aov_v`: 해당 screen pixel이 보고 있는 surface의 UV 좌표

#### 2. Texel-centric gather
- destination texel을 AOV UV round mapping으로 정하지 않음
- `mesh_mask` / `pos3d_tex` 기준으로 texel 후보 선택
- 각 texel에 대해:
  - `pos3d_tex[ty, tx]`를 현재 camera view로 projection
  - projected screen coordinate `(px, py)` 계산
  - image bounds 내부인지 확인

#### 3. Visibility / correspondence 검증
- projected pixel 기준으로 valid 조건 확인
  - `aov_alpha > 0.5`
  - `target_alpha > 0.5`
  - projected pixel의 AOV UV가 현재 texel과 strict nearest 기준으로 일치
- local-neighborhood UV matching은 baseline에서 사용하지 않음

#### 4. Target RGB sampling
- valid texel에 대해 projected floating coordinate `(px, py)`에서 target RGB bilinear sampling
- nearest pixel이 아니라 texel projection 위치의 subpixel coordinate를 사용

#### 5. Texture accumulation
- sampled color를 동일 texel에 기록
  - `accum_np[ty, tx] += sampled_rgb`
  - `weight_np[ty, tx] += 1.0`
- baseline에서는 texel당 view별 최대 1회만 update

#### 6. 중간 texture 저장 및 verify render
- `accum_np / weight_np`로 현재까지의 interim texture 생성
- geometry-space texture를 bitmap/render-space로 vertical flip 후 EXR/PNG 저장
- 갱신된 `interim_texture.exr`로 동일 view를 다시 렌더링하여 post-gather 결과 확인

#### 결과
- 모든 view 처리 후 texel별 평균 계산
  - `baked = accum_np / weight_np`
- `weight_np == 0`인 미커버 texel은 magenta 유지
- main output에서는 nearest-fill, dilation, distance-transform fill 사용하지 않음
- 최종 texture 저장
  - `baked_texture.exr/png`: render용 bitmap-space
  - `baked_texture_geom.exr/png`: UV inspection용 geometry-space
