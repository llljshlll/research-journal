## Summary
- 기존 방식은 view-dependent lighting과 appearance-based mask로 인해 multi-view consistency가 붕괴됨
- 이를 해결하기 위해 lighting을 고정하고, UV visibility 기반 mask를 사용하여 geometry-consistent inverse rendering을 수행함

## 개선 방식

- Lighting 뷰 고정  
  → view마다 다른 illumination이 들어가는 문제를 제거하여 inverse rendering consistency 확보

- UV visibility 기반 mask  
  → shading/texture와 무관하게 실제로 관측된 surface만 업데이트하도록 제한

## 문제점

### 1. 뒷면 생성 오류
- 원인 : diffusion 모델이 prompt에 포함된 "teeth" prior를 강하게 유지하면서, 실제 geometry와 불일치 발생

- 대응 : 뒷면은 별도 prompt 사용 → json에서 뒷면 view 제외 후 진행

### 2. 노이즈 누적
- 원인 : sequential view propagation 과정에서 diffusion error가 누적되며 distribution shift 발생

- 대응 : 뒷면 제외 후, 중심 기준으로 순차 업데이트하도록 json 재구성


### 3. 비가시 영역 업데이트 문제 (gradient leakage)
- 원인 : max_depth = 8 설정으로 인해 카메라에서 보이지 않는 영역까지 gradient가 전달되어 잘못 업데이트됨

- 대응 : 바운딩을 줄이지 않고, 카메라에서 보이는 영역만 업데이트하도록 mask 기반으로 제한

## 카메라 뷰에서 보이는 부분만 수정하도록 바꾼 후 결과

### 설정
- max_depth = 8
- sequential view update
- 각 view마다 inverse rendering 1100~1200 iter 수행
- visibility 기반 mask 적용

### Freeze mask 정의
각 뷰 리파인 시작 시 아래 두 마스크를 결합하여 freeze mask를 구성:

- accumulated_mask  
  → 이전 뷰들에서 직접 hit된 텍셀 누적  
  → 이전 최적화 결과 보호

- (1 - current_coverage)  
  → 현재 뷰에서 보이지 않는 텍셀  
  → max_depth=8에서 발생하는 gradient leakage 차단

### 목적
- 관측된 영역만 업데이트
- 이전 결과 유지 + 비가시 영역 업데이트 방지
