## 기존 방식의 문제

1. **Lighting 문제**
   - 카메라 기준으로 라이트 설정 → 뷰마다 라이트 변경 → view 간 다른 lighting으로 inverse 수행 → consistency 붕괴

2. **Mask 문제**
   - 색상(흰색) 기준 mask → shading 결과와 구분 불가 → 잘못된 영역 포함


## 개선 방식

기준 변경 : 
- Lighting 뷰 고정
- 텍스처/색상이 아니라  **해당 UV가 카메라에 보였는지**로 mask 정의해서 누적 시킴(Mitsuba3의 `aov` integrator와 `aovs: uv:uv`를 사용해 **스크린 픽셀별 UV 좌표를 직접 추출**.)

## 새로운 json파일
(elevation 30°, Azimuth 0°)를 기준으로, 
- 동일 elevation(-30)에서 azimuth를 + 방향으로 15° 간격 확장
- 기준 뷰로 돌아온 뒤 azimuth를 - 방향으로 15° 간격 확장


## json 파일 바꾼 후 결과


바운딩이 8로 설정되어있어, 카메라 뷰에서 안보이는 부분도 수정되어 고정됨
=> 바운딩을 8로 하는 대신, 카메라 뷰에서 보이는 부분만 수정하도록 코드를 수정함 (바운딩을 낮추면, 조명을 이상하게 잡아서 해당 방식으로 수정)

## 카메라 뷰에서 보이는 부분만 수정하도록 바꾼 후 결과
설정 : 바운딩8, 카메라에서 보이는 뷰만 수정, 이전에 수정했던 결과는 수정 안함

각 뷰 리파인 시작 시 아래 두 마스크를 합산해 실제 freeze 마스크 결정: accumulated_mask(이전 뷰들에서 직접 hit된 텍셀 누적 → 이전 최적화 결과 보호) / 1 - current_coverage(현재 뷰에서 안 보이는 텍셀 → max_depth=8 gradient leakage 차단)
