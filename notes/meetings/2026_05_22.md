# Open Model 조사: Semantic Map + Hole Mask 사용 가능 여부

## 판정 기준

- 사용 가능: 공개 pretrained 모델/파이프라인으로 바로 실험 가능
- 학습 필요: 대부분 ontrolNet 구조 또는 학습 코드는 있으나, tooth/gum semantic map용 pretrained 모델은 제한적이라 SD1.5/SDXL을 제외하고는 별도 학습 필요

---

## 요약

| Model | ControlNet | Hole Mask(Inpainting) | ControlNet + Inpainting 함께 사용 | Segmentation ControlNet | 검토결과 |
|---|---|---|---|---|---|
| SD1.5 | O | O | O | O | 사용 가능 |
| SDXL | O | O | O | O | 사용 가능 |
| SD2 | O | O | X | X | 학습 필요 |
| SD3 | O | O | X | X | X |
| FLUX | O | O | O | X | 학습 필요 |
| Qwen-Image | O | O | X | X | X |

---

## 1. SD1.5

- ControlNet + Inpainting 동시 사용 가능, 동시 사용 사례 존재
- SD1.5 segmentation ControlNet 존재
- 공개된 Segmentation ControlNet 존재

링크:
- ControlNet + Inpainting 사용 사례  
  https://github.com/mikonvergence/ControlNetInpaint
- SD1.5 Segmentation ControlNet  
  https://huggingface.co/lllyasviel/control_v11p_sd15_seg
- ControlNet 학습 코드  
  https://github.com/huggingface/diffusers/blob/main/examples/controlnet/train_controlnet.py

검토 결과:
- semantic map + hole mask를 분리 입력해 바로 실험 가능
- 단, pretrained segmentation ControlNet은 일반 segmentation map 기준이므로 tooth/gum semantic map에 대한 반영 정도는 실험에서 확인해야 함

---

## 2. SDXL

- ControlNet + Inpainting 동시 사용 가능
- SDXL segmentation ControlNet 존재
- SD1.5보다 큰 모델이므로 품질은 더 좋을 수 있으나, 실제 tooth/gum semantic 반영 정도는 실험에서 확인해야 함

링크:
- SDXL ControlNet + Inpainting Pipeline  
  https://huggingface.co/docs/diffusers/en/api/pipelines/controlnet_sdxl (StableDiffusionXLControlNetInpaintPipeline)
- SDXL Segmentation ControlNet  
  https://huggingface.co/SargeZT/sdxl-controlnet-seg
- SDXL ControlNet 학습 코드  
  https://github.com/huggingface/diffusers/blob/main/examples/controlnet/train_controlnet_sdxl.py

검토결과:
- 사용 가능
- semantic map + hole mask를 분리 입력해 바로 실험 가능

---

## 3. SD2

- ControlNet 구조 사용 가능
- Inpainting 사용 가능
- image + mask image + segmentation map을 함께 쓰는 공개 pretrained 조합은 확인되지 않음.
- 공개된 SD2용 Segmentation ControlNet도 확인되지 않음

링크:
- ControlNet 학습 코드  
  https://github.com/huggingface/diffusers/blob/main/examples/controlnet/train_controlnet.py
- ControlNet 학습 문서  
  https://huggingface.co/docs/diffusers/main/en/training/controlnet

검토결과:
- 학습 필요
- tooth/gum semantic map을 사용하려면 SD2용 Segmentation ControlNet을 별도로 학습해야 함

---

## 4. SD3

- SD3 ControlNet 구조는 존재
- 그러나 SD1.5/SDXL처럼 ControlNet + Inpainting + Segmentation ControlNet을 바로 조합하는 사용 경로가 명확하지 않음
- 공개된 SD3용 Segmentation ControlNet도 확인되지 않음

링크:
- SD3 ControlNet Pipeline  
  https://huggingface.co/docs/diffusers/api/pipelines/controlnet_sd3

검토결과:
- 현재 목적에서는 바로 사용할 수 있는 경로가 명확하지 않음, 제외

---

## 5. FLUX

- FLUX ControlNet 존재
- FLUX Fill / Inpainting ControlNet으로 hole mask 기반 inpainting 가능
- ControlNet + Inpainting 조합도 가능
- 그러나 공개된 FLUX용 Segmentation ControlNet은 확인되지 않음

링크:
- FLUX ControlNet + Inpainting Pipeline  
  https://huggingface.co/docs/diffusers/api/pipelines/flux#diffusers.FluxControlNetInpaintPipeline (FluxControlInpaintPipeline)
- FLUX ControlNet 학습 코드
  https://github.com/huggingface/diffusers/blob/aa14f090f86c9641abf2da761f39a88133b49f09/examples/controlnet/train_controlnet_flux.py 

검토결과:
- 학습 필요
- hole mask 기반 inpainting은 바로 가능
- semantic map까지 직접 반영하려면 FLUX용 tooth/gum Segmentation ControlNet을 별도로 학습해야 하며, DiT기반 controlNet의 성능 확인도 필요

---

## 6. Qwen-Image

- Qwen-Image ControlNet 존재
- Qwen-Image Inpainting ControlNet 존재
- image + mask image + segmentation map을 함께 쓰는 공개 pretrained 조합은 확인되지 않음.
- 공개된 Qwen-Image용 Segmentation ControlNet도 확인되지 않음

링크:
- Qwen-Image Inpainting ControlNet  
  https://huggingface.co/InstantX/Qwen-Image-ControlNet-Inpainting
- Qwen-Image ControlNet 학습 프레임워크  
  https://github.com/modelscope/DiffSynth-Studio

검토결과:
- 학습 필요
- hole mask 기반 inpainting은 바로 가능
- semantic map까지 직접 반영하려면 Qwen-Image용 tooth/gum Segmentation ControlNet을 별도로 학습해야 함

---

## 최종 정리

현재 pretrained open model 기준으로 semantic map과 hole mask를 분리 입력해 바로 실험 가능한 모델

- SD1.5 Inpainting + Segmentation ControlNet
- SDXL Inpainting + Segmentation ControlNet

FLUX는 ControlNet + Inpainting은 가능하지만, 공개된 Segmentation ControlNet이 확인되지 않았기 때문에 semantic map을 직접 반영하려면 별도 학습이 필요
