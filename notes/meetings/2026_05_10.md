# Strict UV Match → Z-buffer Visibility

## 기존 문제
- 기존 visibility 검증에서는 projected pixel의 AOV UV를 texel index로 round한 뒤, 현재 texel과 strict nearest 기준으로 일치하는지 확인
- 하지만 screen pixel과 texture texel은 해상도와 sampling 기준이 다르기 때문에, 현재 texel을 screen으로 projection한 위치에서 읽은 AOV UV가 현재 texel UV와 정확히 일치하지 않을 수 있음
- 특히 edge, silhouette, 치아 사이, 치아-잇몸 경계에서는 이런 불일치가 더 자주 발생하여, 실제로 보이는 texel도 strict nearest UV match 기준에서는 texture update에서 제외될 수 있음

## 수정 방향
- 기존에는 projected pixel의 AOV UV와 현재 texel UV의 strict nearest 일치 여부를 확인
- 수정 후에는 현재 texel이 해당 screen ray에서 실제로 보이는 표면 위치와 depth 기준으로 일치하는지 확인

## 구현 변경
- 기존 AOV UV 렌더에 depth channel 추가
- AOV 렌더 결과에서 각 screen pixel마다 다음 정보를 저장
  - `aov_coverage`: 해당 pixel이 mesh를 보고 있는지 확인
  - `depth`: 해당 pixel에서 renderer가 실제로 보고 있는 surface까지의 depth 값

## Depth visibility 판정

기존 방식은 projected pixel의 AOV UV가 현재 texel UV와 strict nearest 기준으로 일치하는지 확인하는 방식.

수정 후에는 UV 일치 여부 대신 depth 일치 여부 사용.  
즉, 현재 texel이 projection된 screen 위치에서 실제로 보이는 표면과 같은 depth에 있는지 확인하는 방식.

#### 1. 현재 texel의 screen 위치 계산
- 각 texel의 3D 위치 `pos3d_tex[ty, tx]`를 현재 view로 projection
- projected coordinate `(px, py)` 계산
- `(px, py)`가 image bounds 안에 있는 texel만 후보로 사용

#### 2. AOV depth sampling
- projected coordinate `(px, py)`에서 AOV depth buffer 값을 bilinear sampling
```
sampled_depth = bilinear_sample(depth_buf, px, py)
```
- `sampled_depth`: 해당 screen 위치에서 renderer가 실제로 보고 있는 표면의 depth 값

#### 3. 현재 texel의 expected depth 계산
- 현재 texel의 3D 위치와 camera origin 사이의 거리 계산
```
expected_depth = distance(camera_origin, pos3d_tex[ty, tx])
```
- `expected_depth`: 현재 texel이 해당 view에서 보인다면 가져야 하는 depth 값

#### 4. Depth 비교
- 두 depth 값의 차이가 `depth_eps` 이내인지 확인
```
cond_depth = abs(sampled_depth - expected_depth) <= depth_eps
```
- `cond_depth`: 현재 texel이 해당 screen 위치에서 실제로 보이는 표면과 depth 기준으로 일치하는지에 대한 판정 값

> depth_eps에 대한 자세한 설명은 아래 ## Depth epsilon 참조

#### 5. 최종 visibility 조건
```
visible = cond_aov & cond_tgt_effective & cond_depth
```
| 조건 | 의미 |
|---|---|
| `cond_aov` | projected 위치에서 AOV 기준 mesh coverage 존재 |
| `cond_tgt_effective` | target image에서 색을 가져올 수 있는 상태 |
| `cond_depth` | 현재 texel의 depth와 AOV depth가 `depth_eps` 이내로 일치 |

따라서 현재 방식은 screen에 mesh가 보이는지만 확인하는 것이 아니라, 해당 screen 위치에서 실제로 보이는 표면이 현재 texel과 depth 기준으로 일치하는지까지 확인한 뒤 texture update에 사용하는 구조.

---

## Depth epsilon

`depth_eps`는 `sampled_depth`와 `expected_depth`를 비교할 때 사용하는 허용 오차.

cond_depth = abs(sampled_depth - expected_depth) <= depth_eps

- 너무 작으면 실제 visible texel도 reject되어 edge 미커버가 남을 수 있음
- 너무 크면 서로 다른 표면까지 같은 surface로 accept될 수 있음
- 이 경우 치아-잇몸 경계에서 bleeding 발생 가능

현재 설정은 다음과 같은 구성.

DEPTH_TOL_BBOX_SCALE = 1e-4  
DEPTH_TOL_TEXEL_MULTIPLIER = 0.5  
DEPTH_TOL_MANUAL = None  
   
  
자동 계산식은 다음과 같은 형태  
```
eps_bbox = bbox_diag * DEPTH_TOL_BBOX_SCALE  
eps_texel = robust_texel_spacing * DEPTH_TOL_TEXEL_MULTIPLIER  
depth_eps = max(eps_bbox, eps_texel)
```
| 값 | 의미 |
|---|---|
| `bbox_diag` | mesh 전체를 감싸는 bounding box의 대각선 길이 |
| `robust_texel_spacing` | 인접 texel 간 3D 거리에서 큰 outlier를 제외하고 계산한 대표 texel 간격 |
| `eps_bbox` | mesh 전체 scale을 기준으로 잡은 허용 범위 후보 |
| `eps_texel` | 인접 texel 간 3D 거리를 기준으로 잡은 허용 범위 후보 |
| `depth_eps` | 두 후보 중 더 큰 값을 사용한 최종 허용 범위 |  
  
즉 현재 설정에서는 mesh scale과 texel spacing을 기준으로 depth 비교 허용 오차를 자동 결정하는 방식.
